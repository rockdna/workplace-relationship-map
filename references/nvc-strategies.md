> 斯多葛库管防御、悟空库管心态、存在主义库管意义、反脆弱库管韧性。本库管一件事——**你怎么开口，才能推动事而不是引爆事。**

## 核心工具：长颈鹿语言 vs 豺狗语言

卢森堡用两种动物代表两种沟通模式：

**豺狗语言**（默认模式，99%的职场冲突都在用）：
- 评判："你就是不配合"/"你总是这样"
- 指责："你不尊重我"/"你从不考虑别人"
- 命令："你必须这么做"/"没有商量的余地"
- 比较："别的部门都能做到，就你们不行"
- 否认感受："这有什么好生气的"/"你太敏感了"

**长颈鹿语言**（NVC模式，需要刻意练习）：
- 观察："这周你迟交了3次报告"（不说"总是"）
- 感受："我感到焦虑，因为项目进度受影响"（不说"你不负责"）
- 需要："我需要可预测的交付节奏"（不说"你要配合我"）
- 请求："你愿意每天下班前给我一个进度更新吗？"（不说"你必须每天汇报"）

**关键区别**：豺狗语言指向人（你怎样），长颈鹿语言指向事和需要（我需要什么）。豺狗语言激发防御，长颈鹿语言创造连接。

## 策略一：区分观察与评判

> **来源：马歇尔·卢森堡**《非暴力沟通》第3章

### 第一层｜哲学叩问

卢森堡在书里讲了一个让人后背发凉的观察：**我们以为自己在说事实，其实90%的时候在说评判。**

"你总是迟到"——这是评判，不是观察。观察是"你本周迟到了3次"。
"他不配合"——这是评判。观察是"我发了3封邮件他没有回复"。
"你态度有问题"——这是评判。观察是"你在会上说了3次'这事不归我管'"。

卢森堡会说：评判是豺狗语言的第一步。你一开口就是评判，对方的第一个反应就是防御——不是因为你错了，是因为他的大脑听到的是攻击，不是信息。**你说的是"事实"，他听到的是"你在指责我"。**

职场上：你跟跨部门同事说"你们永远不按时交付"——他听到的是"你在说我能力不行"。但你如果换个方式说"上季度4次交付，2次晚于约定日期"——他听到的是信息，不是攻击。

### 第二层｜策略逻辑

大多数"推不动"的第一步不是策略问题，是**语言模式问题**。你在用评判语言传递信息，对方在用防御模式接收信息——信息传递在语言层面就失败了。

区分观察与评判的核心：**把你嘴里的"总是""从不""太""又"全部替换成具体的数字、时间和事件。**

- "你总是不回我邮件" → "我周一和周三发的两封邮件，目前还没收到回复"
- "他态度太差了" → "他在会上用了3次'这事不归我管'"
- "你从来不提前说" → "上次改方案和这次改方案，我都是在截止当天才收到通知"

**这不是委婉。这是精确。** 委婉是"能不能稍微注意一下"——这还是评判。精确是"3次""2封邮件""截止当天"——这是事实，无法反驳，也不需要反驳。

### 第三层｜具体招式

1. **开口前5秒自检**：你要说的这句话里，有没有"总是""从不""每次""又""太"？如果有——停下来，换成具体数字和时间。
2. **用"我观察到"开头**：把"你不配合"改成"我观察到这周的两次评审，你们部门都没有出席"。
3. **只说行为，不说动机**："他故意不给我信息"是动机揣测。"我在项目群里@了他3次，没有收到回复"是行为观察。你不知道他是不是故意的，但你知道他没回复——只说后者。

**话术（开场）**：
> "我想跟你对一下进度——我观察到上周两次交付晚于约定日期，想了解一下你们这边的情况。"

**风险**：观察不等于回避。有些人把"区分观察与评判"理解成"不能说负面的事"——错。负面事实也是观察。"你迟交了3次"是观察，必须说。只是不要在观察里掺进评判——"你迟交了3次，太不负责任了"，前半句是观察，后半句是评判，删掉后半句。

**🧭 追一问（给用户 2-3 个方向选，至少追两轮）**：

第一轮：
- A. "你现在最想跟对方说的那句'评判'——如果换成'观察'，你会怎么说？"
- B. "你最近一次跟对方沟通时，有没有注意到自己用了'总是''从不''又'这些词？"

第二轮（根据用户第一轮回答追问）：
- 如果A替换不出来 → "试着用这个格式：'在X时间，发生了X次X事'。把人名去掉，只留时间和事件。"
- 如果B说"好像有" → "下次开口前给自己5秒——把那个'总是'换成具体数字。试试看，对方的反应可能完全不同。"

## 策略二：表达感受而非指责

> **来源：马歇尔·卢森堡**《非暴力沟通》第4章

### 第一层｜哲学叩问

卢森堡发现了一个反直觉的事实：**大多数人不会表达感受。**

你以为你在说感受——"你不尊重我""你让我很失望""你太不负责了"——但这些都不是感受。这些是**对对方的评判，伪装成感受**。

真正的感受是："我感到沮丧。""我感到焦虑。""我感到被忽视。"

区别在哪？**评判指向对方，感受指向自己。** "你不尊重我"——主语是"你"，你在说他怎样。"我感到被忽视"——主语是"我"，你在说你怎样。

卢森堡会说：当你用评判伪装成感受时，对方只能听到攻击——因为他被放在了被告席上。但当你真正表达感受时，对方有可能共情——因为每个人都有过沮丧和焦虑。

职场上："你从来不考虑我的时间"——对方听到的是指控。"排期改了3次，我感到很焦虑，因为我的节奏被打乱了"——对方听到的是一个真实的人的状态。

### 第二层｜策略逻辑

职场沟通的默认模式：**用指责代替感受。** 因为表达感受让人觉得"弱"，指责让人觉得"有理"。但指责只能激发防御，感受才能创造理解。

表达感受的策略：**把"你怎样"替换成"我感到"——然后跟上具体的需要。**

- "你不支持我" → "我对这个方向没有获得反馈，我感到不确定"
- "你总是改需求" → "需求改了3次，我感到焦虑，因为我需要可预测的节奏"
- "你抢我功劳" → "我在汇报里没看到我的贡献，我感到被忽视"

**注意**：感受不是示弱。感受是把你的真实状态放在桌面上，让对方看到你是一个人，不是一个对手。这不是软——这是最省力的沟通方式，因为你不需要赢得辩论，你只需要让对方看见。

### 第三层｜具体招式

1. **把"你让我感到"改成"我感到"**："你让我很焦虑"——主语还是"你"，你在怪他。"需求改了3次，我感到焦虑"——主语是"我"，你在说你的状态。
2. **建立你的"感受词库"**：大多数人只会说"不爽""郁闷""烦"。卢森堡建议准备一个具体的感受词库：沮丧、焦虑、困惑、失望、不安、愤怒、无力、委屈、被忽视、被误解。越精确，对方越能理解。
3. **感受后面必须跟需要**：光说"我感到焦虑"不够——对方不知道你要什么。"我感到焦虑，因为我需要可预测的节奏"——现在他知道你要什么了。

**话术（对跨部门同事）**：
> "这周交付又晚了一天，我感到焦虑，因为项目整体进度受影响。我想了解一下你们这边的困难是什么。"

**风险**：在权力不对等的场景下（比如你对老板），表达感受可能被利用——NPD型上司会把"我感到焦虑"当作"他果然不够强"。这时候不是不能表达感受，而是要选择对象和场合。对安全的人表达感受，对不安全的人用观察+请求（不需要暴露感受）。

**🧭 追一问（给用户 2-3 个方向选，至少追两轮）**：

第一轮：
- A. "你最近一次想说但没说出口的感受——是什么？如果用'我感到…'说出来，会怎样？"
- B. "你觉得对方如果听到你的真实感受，会理解还是会利用？"

第二轮（根据用户第一轮回答追问）：
- 如果A说出了感受 → "那下次试试直接说——'我感到XX，因为我需要XX'。说完后看对方反应——你会惊讶有多少人听到感受后会软化。"
- 如果B说"会利用" → "那这个人不适合用感受表达——对他用策略一（观察）和策略四（请求），把感受留给安全的人。"

## 策略三：翻译需要而非猜测策略

> **来源：马歇尔·卢森堡**《非暴力沟通》第5章

### 第一层｜哲学叩问

卢森堡说过一句话，简单到像废话，但99%的沟通冲突都栽在这里：**所有冲突的背后，都是未被满足的需要。**

跨部门不配合——不是他不想配合，是他的需要（比如"我的KPI不在这上面"）没被看见。
老板不表态——不是他不想支持，是他的需要（比如"我需要确保这件事不会翻车"）没被满足。
同事抢功——不是他贪，是他的需要（比如"我需要被看见"）没有被正面满足。

卢森堡会说：**当你只看到对方的"不配合""不支持""抢功"，你在看他的策略——他在用什么方式满足自己的需要。但策略是可以换的，需要是不变的。** 你的任务不是对抗他的策略，是翻译出他的需要——然后找到一种同时满足双方需要的策略。

### 第二层｜策略逻辑

大多数职场沟通失败的原因：**你在讨论策略，不是在讨论需要。**

- 你说"你要配合我"——这是你的策略，不是你的需要。你的需要可能是"可预测性"或"效率"。
- 他说"我这边排不开"——这是他的策略（不配合），背后的需要可能是"对自有资源的控制权"或"不被外部打乱节奏"。

翻译需要的策略：**把双方的策略放一边，先把需要翻出来。** 然后找一种同时满足双方需要的新策略。

- 你需要可预测性 → 他需要对节奏的控制权 → 新策略：每周一固定一个15分钟的对齐会，他在会上提前告诉你这周能排多少资源给你
- 你需要被看见 → 他也需要被看见 → 新策略：联合汇报，各自负责各自的部分

**翻译需要的核心技能：听到"不"的时候，不理解为拒绝，理解为"他有另一个需要在挡路"。**

### 第三层｜具体招式

1. **当对方说"不"时，翻译他的需要**：不要问"你为什么不配合"——问"你这边最需要保障的是什么？"
2. **先说出自己的需要**：不要上来就提策略——先说需要。"我需要项目的关键节点能提前一周确认，因为这样我才能安排测试资源。"
3. **找交集**：把你的需要和他的需要放在桌面上，问——"有没有一种方式，既满足你的XX，也满足我的XX？"

**话术（对不配合的跨部门同事）**：
> "我理解你们这周排不开。我想了解一下——你们这边最需要保障的是什么？如果我们能找到一个既不影响你们节奏、也能让我这边项目不卡住的方式，你愿意试试吗？"

**风险**：翻译需要不等于迁就对方。你是在找一个双方都能接受的方案，不是在单方面让步。如果找不到交集——说明需要真的冲突，那就需要升级决策（找共同的上级），而不是继续僵持。

**🧭 追一问（给用户 2-3 个方向选，至少追两轮）**：

第一轮：
- A. "对方一直不配合——你觉得他真正需要的是什么？不是'他不配合'，是他需要什么？"
- B. "你自己的需要——不是'你要他配合'，是你真正需要什么？可预测性？被尊重？安全感？"

第二轮（根据用户第一轮回答追问）：
- 如果A说"他可能需要XXX" → "那如果有一种方式能满足他的XXX，同时满足你的需要——你愿意跟他谈吗？"
- 如果B说不清自己的需要 → "试着填这个句式：'我需要______，因为______。'不是'我要他怎样'，是'我需要什么'。"

## 策略四：提出请求而非命令

> **来源：马歇尔·卢森堡**《非暴力沟通》第6章

### 第一层｜哲学叩问

卢森堡指出了请求和命令之间一个致命的区别：**如果对方说"不"，你会惩罚他吗？**

如果会——那是命令。如果不会——那是请求。

听起来简单。但职场上，绝大多数"请求"都是伪装的命令：
- "你能不能尽快给我？"（潜台词：不能的话我会追你）
- "希望你能配合一下"（潜台词：不配合的话后果自负）
- "我们需要你的支持"（潜台词：不支持你就不是自己人）

卢森堡会说：**命令激发抵抗，请求创造合作。** 不是因为对方不理性，是因为人的本能——面对命令的第一反应是夺回控制权，哪怕代价是跟命令对着干。

职场上：你跟下属说"你必须这周交付"——他可能交付了，但质量可能敷衍，下次更不愿配合。但你如果问"这周交付有困难吗？如果有的话，哪部分需要调整？"——他反而会更主动。

### 第二层｜策略逻辑

请求的有效性取决于两个条件：
1. **具体**——对方知道你到底要什么
2. **可拒绝**——对方说"不"之后，关系不会破裂

模糊的请求等于没有请求："你能不能注意一下沟通"——注意什么？怎么注意？什么时候？对方听了不知道该做什么。

不可拒绝的请求等于命令："你能不能支持一下？"——如果对方觉得不支持就会被记恨，这就不是请求。

有效请求的格式：**具体动作 + 可执行的时间 + 对方可以说不**

- "你愿意每天下班前花5分钟给我一个文字进度更新吗？如果这个方式不方便，我们可以换一种。"——具体、有时间、可拒绝
- "你愿意在下次评审会上代表我们组汇报这一部分吗？"——具体、可拒绝

### 第三层｜具体招式

1. **把"希望""能不能""尽量"全部替换成具体的"你愿意…吗？"**：模糊是命令的掩护。"你能不能配合"是命令的温柔版。"你愿意每周二下午花15分钟跟我对一下进度吗？"是请求。
2. **每次请求后面加一个"出口"**：让对方知道说"不"是安全的。"如果这个时间不方便，我们可以找另一个时间。"——这不是让步，是让对方知道这是真正的请求。
3. **请求被拒后，翻译需要**（回到策略三）：他拒绝了——不是他不合作，是他有另一个需要。问："你这边最大的顾虑是什么？"

**话术（对老板）**：
> "我想确认一下方向——你愿意这周五之前给我一个明确的'做还是不做'吗？如果还需要更多信息才能决定，我可以这周补一份简版方案给你。"

**风险**：在权力不对等的场景下，"请求"可能被视为"不够坚定"。对老板提请求时，语气要稳，不是试探——是"我需要这个信息才能推进，你看什么时间可以"。不是"你能不能给我一个回复"（软），是"我需要一个明确的方向，你看周五前可以吗"（稳）。

**🧭 追一问（给用户 2-3 个方向选，至少追两轮）**：

第一轮：
- A. "你现在最想让对方做的那件事——如果用'你愿意…吗？'说出来，会怎样？"
- B. "你上一次提'请求'——对方是觉得你在请求，还是觉得你在下命令？"

第二轮（根据用户第一轮回答追问）：
- 如果A说不出具体请求 → "试着填这个句式：'你愿意在X时间做X事吗？如果不行，我们可以调整。'把X填上。"
- 如果B说"可能是命令" → "那说明你的'请求'没有给对方说不的出口。下次在请求后面加一句：'如果这个方式不方便，我们可以想另一种。'"

## 策略五：接住对方的豺狗

> **来源：马歇尔·卢森堡**《非暴力沟通》第7-9章 + 《用非暴力沟通化解冲突》

### 第一层｜哲学叩问

卢森堡讲了一个让大多数人不敢试的策略：**当对方用豺狗语言攻击你时，不要还击，也不要沉默——翻译他。**

他说"你就是不配合"——你翻译：他在说他的需要没被满足。
他说"你们部门就知道推事"——你翻译：他在说他感受到了压力。
他说"你到底懂不懂"——你翻译：他在说他需要被理解。

卢森堡不是说对方有理。他说的是：**豺狗语言是未被翻译的痛苦。** 对方用攻击表达，是因为他不知道怎么用感受和需要表达。你的任务不是接住攻击——是绕过攻击，翻译他背后的话。

职场上：老板在会上说"这事怎么做的一塌糊涂"——你不用辩护（防御），也不用认错（屈服），你翻译："我听到您对当前结果不太满意——您最关心的是哪个方面？"

### 第二层｜策略逻辑

接住豺狗的策略分三步：

1. **不接攻击，只接信息**：对方说"你太不负责了"——"不负责"是评判，你不接。你接他话里的信息：他有一个需要没被满足。
2. **翻译成感受+需要**："听起来你对交付结果感到不满——你这边需要的是什么？"
3. **再提出你的请求**：翻译完之后，再回到你的需要和请求。

这不是忍让。这是**降维打击**——对方用攻击模式，你用翻译模式。攻击需要对手才能继续，翻译让攻击失去目标。他打空气。

### 第三层｜具体招式

1. **当对方用评判语言时，用"听起来…"开头翻译**：
   - "你就是不配合" → "听起来你这边有些困难——你目前最需要的是什么？"
   - "你们永远不提前说" → "听起来你对信息同步有期待——你希望在什么时间节点收到通知？"
2. **不要纠正对方的语言**：对方说"你太不负责任了"——不要说"你不能这么说，这是评判"。那是说教。只翻译，不纠正。
3. **翻译完之后，再表达你的需要**：先接住他，再回到你。"我理解你这边有压力。同时，我需要项目节点能提前确认——你愿意这周我们定一个双方都能接受的时间表吗？"

**话术（对发火的老板）**：
> "我听到您对当前进度不太满意。您最关心的具体是哪个方面？——我这边需要知道优先级，才能调整下一步。"

**风险**：接住豺狗不等于接受侮辱。如果对方的攻击已经超出职业底线（人身攻击、歧视性语言），NVC不是让你继续翻译——是让你设边界。"我愿意讨论工作问题，但我需要对话保持在职业范围内。"设边界本身也是NVC——你在表达你的需要（被尊重）。

**🧭 追一问（给用户 2-3 个方向选，至少追两轮）**：

第一轮：
- A. "对方最近一次用豺狗语言攻击你——他说的那句话，如果翻译成'感受+需要'，是什么？"
- B. "你觉得如果你用'听起来…'翻译他的话，他会怎么反应？"

第二轮（根据用户第一轮回答追问）：
- 如果A翻译出来了 → "下次他再说类似的话，试试直接翻译——'听起来你感到XX，你这边需要的是XX？'看看他的反应。"
- 如果B说"他会更生气" → "第一次可能确实会——因为攻击没打中目标，他会困惑。但困惑比愤怒好。困惑意味着他开始想'这个人怎么没按套路出牌'——这时候你再说你的需要，他反而听得进去。"

## 策略六：自我同理先行

> **来源：马歇尔·卢森堡**《非暴力沟通》第9章

### 第一层｜哲学叩问

卢森堡在书的最后讲了一个容易被忽略的前提：**你不可能对别人使用NVC，如果你对自己都没有同理心。**

什么意思？当你带着愤怒、委屈、不甘走进一场对话——你嘴里说的是"我观察""我感到"，但你的身体语言、语气、微表情全都在说"我是对的，你是错的"。对方不是听你说什么——对方是读你的状态。

卢森堡会说：**先给自己3分钟。** 不是压抑情绪，是承认它——"我现在很愤怒。我愤怒是因为我的需要（被尊重/被看见/公平）没被满足。"承认完之后，愤怒不会消失，但它会从"我要搞他"变成"我需要处理我的需要"。

职场上：你被抢功了，你冲去找对方理论——这时候你嘴里说的每一个"我观察""我感到"，对方都能听出你在算账。但如果你先给自己5分钟，承认"我很愤怒，我需要被看见"——然后带着这个清醒进去，你说的话会完全不同。

### 第二层｜策略逻辑

自我同理不是情绪管理课上的"深呼吸"。是**先把自己从"战斗模式"切换到"需要模式"**。

战斗模式："他抢我功劳，我要讨回公道。" → 你的策略都是对抗性的
需要模式："我的贡献没被看见，我需要被承认。" → 你的策略可以是建设性的

同样一个困局，两个模式给出的策略完全不同。战斗模式让你去找对方对质，需要模式让你去找一个让贡献被看见的方式——可能是在下次汇报时主动认领自己的部分，可能是跟老板一对一澄清，可能是联合汇报。

**自我同理的核心：先问自己"我现在的感受是什么，背后的需要是什么"——再决定做什么。**

### 第三层｜具体招式

1. **开口前问自己三个问题**（不需要3分钟，30秒就够）：
   - 我现在什么感受？（愤怒/委屈/焦虑/失望——选一个最准确的词）
   - 这个感受背后的需要是什么？（被尊重/被看见/公平/安全感——选一个）
   - 我现在开口，是在满足这个需要，还是在发泄这个感受？
2. **如果答案是"在发泄"——停下来**。不是说不能发泄，是说发泄不会满足你的需要。你需要的是被看见，发泄让对方看见的是你的愤怒，不是你的贡献。
3. **把"他怎样"全部换成"我需要什么"**："他抢我功劳"→"我需要我的贡献被看见"。"他不支持我"→"我需要方向上的确定性"。需要清楚了，策略就清楚了。

**话术（给自己）**：
> "我现在很愤怒。我愤怒是因为我的贡献没被承认——我需要被看见。被看见的方式不一定是跟他吵，也可以是下一次汇报时我自己讲。"

**风险**：自我同理不等于自我合理化。"我感到愤怒是因为他太过分了"——这是合理化，不是自我同理。自我同理是"我感到愤怒，我需要公平"——指向自己的需要，不指向对方的错。

**🧭 追一问（给用户 2-3 个方向选，至少追两轮）**：

第一轮：
- A. "你最近一次带着情绪开口——如果先给自己30秒做自我同理，你猜结果会不同吗？"
- B. "你现在的感受是什么？背后的需要是什么？"

第二轮（根据用户第一轮回答追问）：
- 如果A说"可能不同" → "那下次开口前，先花30秒问自己三个问题。写下来也行——感受、需要、我现在开口是在满足需要还是在发泄。"
- 如果B说出了感受和需要 → "好，需要清楚了。现在问自己：满足这个需要的方式有哪些？最少想3个——第一个通常是'搞他'，跳过，看第二个和第三个。"


## 何时用哪一策

```
你的困局是什么？
│
├─ 开口就是"你总是""你从不"，对方听了就炸 → 区分观察与评判
├─ 想表达不满但只会指责，对方只听到攻击 → 表达感受而非指责
├─ 对方不配合，你说服不了他 → 翻译需要而非猜测策略
├─ 你提的"请求"对方觉得是"命令" → 提出请求而非命令
├─ 对方用攻击语言，你不知道怎么接 → 接住对方的豺狗
└─ 带着情绪开口，说完就后悔 → 自我同理先行
```

## 五库路由总则（斯多葛库 + NVC库 + 悟空库 + 存在主义库 + 反脆弱库）

```
用户困局 → 先判断层级
│
├─ 防御层：「他在控制我 / 我开始怀疑自己」→ 斯多葛策略库
│   特征：煤气灯、被否定、叙事入侵、情绪勒索、自我怀疑
│
├─ 沟通层：「我说了他不听 / 推不动 / 不知道怎么开口」→ NVC策略库
│   特征：跨部门推不动、老板不表态、同事冲突、需要正面交涉
│
├─ 心态层：「我被困住了」→ 悟空策略库
│   特征：关系纠缠、权力斗争、身份困惑、情绪内耗
│
├─ 系统层：「我该怎么办？」→ 反脆弱策略库
│   特征：不确定性、风险、安全、退路、方向选择
│
└─ 意义层：「我做这些是为了什么？」→ 存在主义策略库
    特征：麻木、迷茫、焦虑、自由恐惧、他人评价
```

**交叉信号**：用户可能同时需要多个库。比如"跨部门同事不配合，我很焦虑"——这个困局涉及：
- 沟通层（怎么开口才能推动他）→ NVC库：翻译需要
- 系统层（如果沟通也不行，退路在哪）→ 反脆弱库：可选性囤积
- 心态层（被反复推不动搞得情绪内耗）→ 悟空库：心猿归正

**这时候按优先级出：先解决沟通问题（NVC），因为大多数"推不动"只是语言模式错了。如果NVC用完还是推不动，说明不是沟通问题，是结构问题——再上反脆弱。如果连带着情绪内耗——再上悟空。**

**防御层优先原则**：如果用户同时出现自我怀疑/叙事入侵（斯多葛信号）和沟通困难（NVC信号）——先建防御（斯多葛），再学沟通（NVC）。自我怀疑不解决，NVC也用不好——你会用"我感到"开头，但你的身体语言全是攻击。
