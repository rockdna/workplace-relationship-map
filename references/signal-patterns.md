# 阻力信号识别规则 v2.1

从用户描述中识别阻力信号的规则库。v2.1 新增：被动排除、定位固化（2 种新信号类型，覆盖被动遗忘和角色固化的高频盲区）。

---

## 十类核心阻力信号

### 1. 怕担责（Risk Aversion）

**典型表述**：
- "出了事谁负责？"
- "这个方案之前试过，后来出问题了"
- "我需要再请示一下" / "等我再想想"
- "这个事情比较敏感"

**隐性信号**（用户可能不会直接说，但从行为推断）：
- 该签字的拖了三天没签
- 在会上不表态，会后私下说"我觉得可以"
- 把决定推给上级或会议

**真实顾虑**：不是"方案好不好"，是"出事了谁背锅"。

**突破口**：先给退路，再给方案。不要先说"这个方案多么好"，先说"如果出了问题，有X、Y、Z兜底"。

**置信度判断**：如果用户描述中有"他之前因为类似的事受过批评/处分"，高置信度。

---

### 2. 利益冲突（Interest Conflict）

**典型表述**：
- "这样做会增加我们团队的工作量"
- "对我们的KPI没有帮助"
- "这件事不在我们的OKR里"
- "你们能不能自己搞定？"
- "这个方案对我们来说成本太高"

**隐性信号**：
- 对方突然开始问"这个方案的ROI是多少"——其实不是真关心ROI，是在找拒绝理由
- 对方把讨论引向"资源问题"——其实资源够，他只是不想给
- 对方提了一堆条件但每个都是"能不能再看看"——他在谈判，不是在拒绝

**真实顾虑**：这件事对他没有直接收益，甚至有成本。

**突破口**：把"你需要配合"翻译成"这对你有什么好处"。不是利诱，是让他看到自己也能得到什么。关键是找到他的KPI/OKR里这件事对应的条目。

**置信度判断**：如果对方主动提到KPI/OKR/工作量，高置信度。

---

### 3. 历史创伤（Past Trauma）

**典型表述**：
- "上次类似的事最后不了了之了"
- "我们之前试过，没什么用"
- "我当时提过意见，但没被采纳"
- "又来？上次也说要做"

**隐性信号**：
- 对方一听到你的方案就叹气/冷笑
- 对方开始讲上次的事——看似跑题，其实在说他为什么不愿意
- 对方说"只要不跟上次一样就行"——他不是在提要求，是在给自己安全感

**真实顾虑**：不是反对当前方案，是在保护自己不再经历上次的挫败。

**突破口**：承认历史，不要否定他的经历。"我知道上次没成，这次我们做了三件事来避免同样的问题"。不是"这次不一样"，是"这次我们想到了上次没考虑到的东西"。给出具体的差异点，而不是笼统的安慰。

**置信度判断**：如果用户提到"上次/之前/以前"，高置信度。

---

### 4. 信息不对称（Information Gap）

**典型表述**：
- "我不太了解具体情况"
- "你们之前是怎么做的？"
- "能给我看看详细方案吗？"
- "这个方案的背景是什么？"

**隐性信号**：
- 对方在会上一直沉默——不是不同意，是听不懂
- 对方提了很多基础问题——他不是在质疑，是在搞懂
- 对方说"让我再消化一下"——他是真的需要时间理解

**真实顾虑**：不是不想配合，是根本不知道你在干什么。他的犹豫是在争取时间搞懂情况。

**突破口**：先花5分钟讲清楚来龙去脉，不要上来就谈合作。信息到位了，阻力自然降。最有效的做法：在正式会议前，先一对一聊10分钟背景。

**置信度判断**：如果对方的问题都是"是什么/怎么做"而不是"为什么/凭什么"，高置信度。

---

### 5. 优先级冲突（Priority Conflict）

**典型表述**：
- "我这周真的忙不过来"
- "等我处理完手头这个项目"
- "下个月再看吧"
- "能不能先简单做个MVP？"

**隐性信号**：
- 对方总是"临时有事"取消会议——你的事在他的日程里排不上
- 对方一直催你给简化版——他想做，但不想花太多精力
- 对方说"你先推，后面我来接"——他在降低自己的投入

**真实顾虑**：不是排斥你的方案，是手头有更紧急的事。你的优先级不是他的优先级。

**突破口**：把"你需要现在配合"变成"这件事什么时候配合最合适"。给他一个他日程里能装下的窗口，而不是催他马上做。如果能帮他解决一个他更紧急的问题作为交换，效果最佳。

**置信度判断**：如果对方说"忙"但还在跟你讨论细节（而不是完全不回复），说明他有兴趣只是没时间——高置信度。

---

### 6. 权力姿态（Power Posturing）

**典型表述**：
- "这个事情我来定" / "我觉得应该这样做"
- 在会议上质问、打断、纠正
- "你们的方案我看了，有几个问题"
- 对你的方案逐条挑刺但自己不给方案

**隐性信号**：
- 对方只找你一个人的问题，不找别人的——他在确立对你的优势
- 对方在公开场合质疑但私下不沟通——他要的是"在场的人看到他比你强"
- 对方的"建议"都是命令式语气——不是在讨论，是在宣布

**真实顾虑**：不是方案的问题，是他需要确认自己在权力结构中的位置。他怕被你或者你的方案"盖过"。

**突破口**：不要接他的框架。他质疑你的方案，你不用在方案层面回应——你需要在关系层面回应。最有效的方式：在公开场合认可他的地位，在私下推进你的内容。即"公开给面子，私下走事情"。

**置信度判断**：如果质疑只发生在有第三方的场合，高置信度。

---

### 7. 忠诚测试（Loyalty Test）

**典型表述**：
- "你跟谁更熟？你先找他还是找我？"
- "这件事你汇报给谁了？"
- "你跟XXX什么关系？"
- "你怎么看XX部门那个方案？"

**隐性信号**：
- 对方在意你"先找谁"而不是"说了什么"——他在测试你的站队
- 对方问你对另一个人的看法——他在判断你属于哪个阵营
- 你同时收到两边的会议邀请——两边都在抢你的态度

**真实顾虑**：他不是在拒绝你，是在确认"你是不是我的人"。在他确认之前，他不会支持你。

**突破口**：不要回答立场问题，要回答信任问题。不是"我站你这边"，而是"你看，我一直在跟你同步进展"。让他看到你的行为证明你的倾向，而不是语言表态。语言表态太廉价，而且可能传到另一方耳朵里。

**置信度判断**：如果组织正在经历变革/领导更替，高置信度。

---

### 8. 战略性拖延（Strategic Delay）

**典型表述**：
- "我再研究研究"
- "让我再拉几个人一起看看"
- "我们需要更充分的论证"
- "这个决定不能急"

**隐性信号**：
- 拖了三周还在"研究"但什么研究都没做——他不是在研究，是在等
- 他提出要"再拉几个人"——他在稀释你的推动力，或者找自己人进来平衡
- 每次你问进展他都说"快了快了"——他根本没在推

**真实顾虑**：他不想做这件事，但又不想直接说"不做"。拖到你放弃，或者拖到这件事自己过时。

**突破口**：给一个不可逆的时间节点。"老板要在XX号前看到结果，不管做什么决定，那天之前必须有结论。" 让他意识到"拖延本身也是一种决定，而这个决定的后果他也要承担"。

**置信度判断**：如果对方每次都有新的"需要再看看"的理由但不推进任何实质动作，高置信度。

### 9. 被动排除（Passive Exclusion）

**典型表述**：
- "会议没叫我了"
- "项目被换了不通知我"
- "我问了才说忘了"
- "团建时大家聊项目我插不上话"

**隐性信号**：
- 对方说"忘了跟你说"——不是撒谎，是你在他的通知列表里真的不在前面
- 你的产出被「部门」统一概括而不是单独列出——个体被模糊成集体
- 没有人主动反对你，但也没有人主动找你——你从工作流里"自然蒸发"了

**真实顾虑**：不是有人故意排挤你，是默认流程里你已经被绕过。没有敌人可以对抗——正是因为没有敌人，才更难翻身。

**突破口**：不要等人想起你——自己做一件他们不得不讨论的事。重返流程只有一个办法：创造一个不可忽视的入口。在周会上直接展示方案，而不是先等组长批准。

**置信度判断**：如果没有任何人公开反对你、但你就是接不到新任务，且持续超过 1 个月——高置信度。

**与信息不对称的区别**：信息不对称是「对方不了解你在做什么」，被动排除是「对方的工作流里已经没有你的位置」。前者可以通过沟通解决，后者必须通过**重新插入流程**来解决。

---

### 10. 定位固化（Role Fixation）

**典型表述**：
- "你做得很好，但是……"（后面跟的是跟你干得差不相关的事）
- "你在这个位置上很稳定"
- 老板给你分配的都是同类任务，从不给你新方向的机会
- "我们需要你在现在的岗位上继续发光"

**隐性信号**：
- 老板对你的评价永远是"靠谱、稳定、让人放心"——但不是"有潜力、能挑更重的担子"
- 你提新方向时他说"先把现在的做好"——他不是否定你的能力，是否定你「不属于那个方向」
- 你的晋升停滞不是因为绩效不好——是因为你被贴了「执行者」标签，老板看不到你做「策略者」的样子

**真实顾虑**：不是老板不认可你——是他对你的定位已经固化了。你的「好用」反而锁死了你的发展路径。

**突破口**：停止「什么活都做好」，开始「只做跟下一步目标匹配的活」。让老板看到你在新方向上的产出，而不是听说你想转方向。关键不是表态，是用作品重新定义你的标签。

**置信度判断**：如果连续 2 个考核周期你的工作内容没有实质变化且晋升被绕过——高置信度。

**与优先级冲突的区别**：优先级冲突是「对方手头有更紧急的事」，定位固化是「对方认为你只适合做这一类事」。前者是时间问题，后者是认知问题。

## 信号组合决策树

当多种信号同时出现时，优先处理主信号：

```
用户描述
    │
    ├─ 对方有明确反对动作？── 是 → 利益冲突 或 权力姿态
    │                           │
    │                           ├─ 反对发生在公开场合？→ 权力姿态（主）
    │                           └─ 反对只在私下？→ 利益冲突（主）
    │
    ├─ 对方没有反对但也没有行动？── 是 → 优先级冲突 或 战略性拖延
    │                                │
    │                                ├─ 他在做别的事？→ 优先级冲突
    │                                └─ 他什么都没做？→ 战略性拖延
    │
    ├─ 对方问了你的立场？── 是 → 忠诚测试
    │
    ├─ 对方提了上次的事？── 是 → 历史创伤（主）+ 可能叠加 怕担责
    │
    └─ 对方问了很多基础问题？── 是 → 信息不对称
```

## 识别技巧

1. **听他说的话，但看他为什么这么说**。同是"我需要再想想"，可能是怕担责，可能是优先级冲突，可能是历史创伤。看语境。
2. **找他的"但是"**。但是后面才是真话。前面都是礼貌。
3. **看他做了什么，不是说了什么**。嘴上说"没空"，实际是不想让你推进。嘴上说"需要再看看"，实际是在等别人先表态。
4. **阻力最大的那个人，通常是上一次类似事件中受损最多的人**。
5. **如果对方的态度突然变化（之前配合突然不配合），一定是发生了你不知道的事**——去搞清楚。
6. **"不回复"本身就是一个信号**——它不是"没有态度"，它是"我不想表态"。
